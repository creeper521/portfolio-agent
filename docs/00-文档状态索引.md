## 1. 当前实现基线

当前仓库已经交付：

- 一个 SQL 审计项目、一个规范问题、一组已审核 Evidence 和一条 TimelineEvent；
- 六个正式前端路由、四层首页、Agent 三栏工作台和仅限当前页面内存的临时会话；
- `GET /api/v1/public-content` 聚合接口、服务端 QuestionPreset、四维 Answer API 契约和结构化 sections；
- 一次性内存 handoff、ANSWERED/BOUNDARY/REJECTED、单 JAR 打包与 packaged-JAR Playwright 联调；
- 默认关闭的 C1 模型表达：白名单 AnswerPlan、DeepSeek V4 Flash/GLM-4.7 单 Provider、完整 Draft 校验和同 Plan fallback；
- C2a 本地公开检索：BGE-small-zh-v1.5 INT8 ONNX、发布期文档向量、运行期本地查询向量、BM25/向量/RRF/Grounding Gate 与七文件原子 Bundle；
- C2b 固定只读工具与引用式多轮：六类封闭 ToolKind、同 Snapshot 执行、稳定引用重验证、页面内存 ContextEnvelope 与版本变化提示；
- `common / portfolio / answer` 模块化单体、代码质量/架构/隐私/静态 bundle 发布门禁。

逐功能证据、受限条件、未实现范围和建议优先级见 `docs/08-current-implementation-status.md`。

当前仍不是完整 V1。C1 未引入 Spring AI；C2 已完成本地公开检索、固定只读工具和引用式多轮。C3 中仅 Model Provider Registry 已实现：不可变快照 `c3-model-registry-v1` 内建 DeepSeek V4 Flash 与 GLM-4.7，沿用既有环境变量、每次部署显式选择一个 Provider，且没有自动故障转移或动态发现。Tool Registry、Hook、Orchestrator、多 Agent、DurableTask、持久会话、数据库、认证和私有 Copilot 均未准入、未实现。

## 4. 当前已知但尚未关闭的状态差异

1. C1 已实现但默认关闭；真实部署是否启用仍取决于部署方的数据策略审批与运行时密钥。C2 本地检索默认关闭；固定只读工具和引用式多轮已实现。C3 中仅 Model Provider Registry 已实施，快照固定为 `c3-model-registry-v1`，内建两个 Provider、每个部署显式选择一个，且无自动故障转移或动态发现；Tool Registry、Hook、Orchestrator、多 Agent、DurableTask 和持久会话仍未准入、未实施。
2. 当前公开内容仍只有一个可执行 QuestionPreset；页面只展示该真实支持问题，不再展示不可执行的角色问题。
3. B 的 bootstrap bundle 是经过自动化契约验证的开发/首装种子，不代表一次真实生产人工 Approval；真实内容发布必须由明确的人类审核者在外部私有工作区重新批准。
4. 完整发布脚本会执行全套门禁；直接 `mvn package` 仍只代表构建与后端测试成功，不能替代完整发布验证。

下一阶段不是自动扩展 C3。C2 已固定本地 Embedding、索引、只读 ToolPlan 和引用式多轮契约；访客问题、查询向量、检索信息和历史问答仍不得发送给外部 Provider。Model Provider Registry 已限定为两个内建 Provider、单 Provider 显式选择和不可变快照；只有其余 C3 的真实准入条件满足并单独形成 ADR 后，才可讨论 Tool Registry、通用 Hook、Orchestrator、多 Agent 或 DurableTask。
