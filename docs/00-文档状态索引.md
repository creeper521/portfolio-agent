## 1. 当前实现基线

当前仓库已经交付：

- 一次性内存 handoff、ANSWERED/BOUNDARY/REJECTED、单 JAR 打包与 packaged-JAR Playwright 联调；

当前仍不是完整 V1，也没有实现 DeepSeek、Spring AI、RAG、Embedding、SSE、数据库、认证、服务端会话、动态外部发布或私有 Copilot。

## 4. 当前已知但尚未关闭的状态差异

1. C 尚未实施：没有模型表达、Embedding、公开检索、只读工具、引用式多轮、Registry、Orchestrator 或多 Agent；具体模型也尚未选型。
2. 当前公开内容仍只有一个可执行 QuestionPreset；页面只展示该真实支持问题，不再展示不可执行的角色问题。
3. B 的 bootstrap bundle 是经过自动化契约验证的开发/首装种子，不代表一次真实生产人工 Approval；真实内容发布必须由明确的人类审核者在外部私有工作区重新批准。
4. 完整发布脚本会执行全套门禁；直接 `mvn package` 仍只代表构建与后端测试成功，不能替代完整发布验证。

下一阶段须先向用户汇报 C1 模型表达候选方案和权衡并获得确认；不得把 C1/C2/C3 写成已完成，也不得提前接入 Provider、RAG、Registry 或多 Agent。
