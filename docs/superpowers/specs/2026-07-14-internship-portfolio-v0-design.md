# 实习作品集 Agent V0 纵向切片设计

> 状态：已实施，是当前 V0 事实与确定性回答边界
> 日期：2026-07-14
> 依据：`docs/01-项目背景.md`、`docs/02-需求探索文档.md`、`docs/03-可能技术选型.md` 及本轮 Superpowers brainstorming 决策

## 1. 设计目标

V0 用一个真实项目打通公开作品集 Agent 的最小端到端链路。它不是纯页面原型，也不是完整 V1。

目标访问者是技术面试官和实习导师。访问者应能从首页进入 SQL 审计与故障排查工具详情页，并通过一次对话了解项目背景、个人职责、技术方案、验证过程和最终状态，同时看到经过脱敏审核的 Evidence 卡片。

V0 的产品成功标准：陌生访问者不阅读私有日报，也能准确理解该项目的价值、本人贡献边界和交付状态。

## 2. 已确认决策

- V0 只展示 SQL 审计与故障排查工具项目。
- 提供最小首页、一个项目详情路径、一个自由输入的对话入口和一张 Evidence 卡片。
- 唯一稳定支持的规范问题为：

  `请详细介绍 SQL 审计与故障排查工具项目：背景、我的职责、技术方案、验证过程和最终状态分别是什么？`

- 允许少量人工配置的等价问法。其他问题明确返回当前能力边界和推荐问题。
- V0 不调用 DeepSeek，不使用 Spring AI 运行链路，不使用 SSE。
- 回答使用普通 REST 一次性返回，并在协议中保留后续模型与 fallback 扩展字段。
- 数据使用知识库真实内容整理出的首份脱敏公开 JSON 快照。
- Evidence 使用“SQL 审计工具交付证据集”组合卡片，不公开日报原文、内部截图和内部路径。
- 项目状态为 `DELIVERED`，贡献类型为 `PRIMARY`。
- 对外状态说明：核心版本已完成、部署并形成使用文档；本人主导核心功能及多轮迭代，后续部分优化由同事继续接手。
- 采用端到端纵向切片，而不是前端 Mock 优先或后端孤立实现。

## 3. 产品范围和用户路径

### 3.1 首页 `/`

首页展示：

- 站点定位和实习角色。
- 一段简短个人简介。
- SQL 审计项目卡片。
- 项目状态和贡献类型。
- “查看项目”入口。

真实姓名、求职邮箱、GitHub 和简历 URL 尚未在权威文档中给出。公开快照允许这些字段为空；为空时界面不渲染对应元素，不使用虚构值。

### 3.2 项目详情 `/projects/sql-audit`

详情页展示：

- 项目背景和业务问题。
- 本人职责及贡献边界。
- 核心技术方案和关键设计决策。
- 验证方式与交付结果。
- 当前状态和后续交接说明。
- 内嵌 Agent 对话区。

详情页不创建独立聊天路由，避免 V0 出现只有聊天框、缺少可浏览内容的问题。

### 3.3 对话行为

- 输入框允许自由输入，并提供规范问题的推荐按钮。
- 命中规范问题或人工配置的等价问法时，返回五段式确定性回答。
- 五段分别为背景、职责、方案、验证、状态。
- 回答后展示 Evidence 卡片。
- 未命中时返回能力边界说明和唯一推荐问题。
- 不暗示 V0 可以回答全部实习问题，不模拟生成式智能。

## 4. 视觉与交互设计

### 4.1 Design Read

Reading this as: 面向技术面试官和实习导师的开发者作品集，采用专业、证据导向、克制但有辨识度的视觉语言，使用 Vue 与原生 CSS 变量构建轻量自有样式。

设计参数：

- `DESIGN_VARIANCE = 6`
- `MOTION_INTENSITY = 3`
- `VISUAL_DENSITY = 4`

### 4.2 视觉原则

- 使用单一冷调蓝绿色作为强调色，避免 AI 紫色渐变。
- 使用统一的 14px 卡片圆角、10px 控件圆角和全圆角状态标签。
- 采用非对称但清晰的桌面布局，移动端在 768px 以下折叠为单列。
- 字体优先使用系统中文无衬线字体，避免为了字体效果引入外部阻塞资源。
- 页面支持系统浅色和深色模式，统一使用 CSS 语义变量。
- 首页首屏控制在一个视口内，CTA 无需滚动即可看到。
- 不制作虚假产品截图，不公开未经审核的内部截图。
- 使用内容结构、代码感排版和 Evidence 索引作为真实视觉素材。

### 4.3 状态和可访问性

- 提供页面加载骨架、接口错误、问题未命中、提交中和成功状态。
- 输入框必须有可见标签，不以 placeholder 代替标签。
- 支持键盘提交，焦点样式清晰。
- 按钮、正文、占位符和错误信息满足 WCAG AA 对比度。
- 动效只用于页面进入和交互反馈；系统偏好减少动画时全部禁用。
- 不使用逐帧滚动监听、复杂视差和无限循环动画。

## 5. 系统架构

项目采用同仓库前后端分层结构：

- `frontend/`：Vue 3、TypeScript、Vite、Vue Router。
- `backend/`：Java 21、Spring Boot、Maven。
- `backend/src/main/resources/public-data/public-portfolio.v1.json`：唯一公开事实快照。

开发环境中前后端分别启动，Vite 将 `/api` 代理到 Spring Boot。生产构建时，Maven 在构建阶段调用前端构建并把静态产物放入 Spring Boot 可执行 JAR，最终只部署一个应用制品。

后端调用链：

1. `PortfolioController` 接收只读请求。
2. `PortfolioQueryService` 编排项目查询和问答。
3. `QuestionNormalizer` 规范化问题并进行有限匹配。
4. `AnswerEngine` 定义回答边界。
5. `DeterministicAnswerEngine` 根据公开快照组织回答。
6. `PublicPortfolioRepository` 提供启动时加载的只读数据。

`AnswerEngine` 保留后续接入 DeepSeek 的扩展点。V0 仅注册确定性实现，不提前引入无运行价值的模型依赖。

## 6. 公开快照模型

顶层字段：

- `schemaVersion`
- `contentVersion`
- `publishedAt`
- `owner`
- `projects`
- `questions`
- `evidence`

`owner` 包含姓名、实习角色、简介以及可为空的 GitHub、邮箱和简历 URL。

`Project` 包含：

- `id`、`slug`、标题和摘要。
- 背景、职责、方案、关键决策、技术栈。
- 验证方式、交付结果和交接说明。
- `status`、`contributionType`。
- 关联的问题 ID 和 Evidence ID。

`Question` 包含：

- 问题 ID和项目 ID。
- 规范问题。
- 人工审核的等价问法。
- 推荐问题展示文本。

回答不保存第二份事实文案，而是按固定模板读取 Project 字段组织，保证详情页与对话事实一致。

`Evidence` 包含：

- Evidence ID和标题。
- 类型 `COLLECTION`。
- 覆盖时间范围和来源数量。
- 脱敏摘要和所支撑的结论。
- 公开状态 `APPROVED`。
- `rawContentPublic = false`。

公开快照不得包含本地知识库路径、内部文档文件名、服务器 IP、账号、目录、凭据、内部系统地址、数据库内部结构、未审核截图、日报原文和内部人员信息。

## 7. API 契约

### 7.1 `GET /api/v1/portfolio`

返回首页使用的个人公开信息和项目摘要。

### 7.2 `GET /api/v1/projects/{slug}`

返回指定项目详情、Evidence 摘要和推荐问题。不存在时返回 404。

### 7.3 `POST /api/v1/answers`

请求：

```json
{
  "projectSlug": "sql-audit",
  "question": "请详细介绍 SQL 审计与故障排查工具项目……"
}
```

响应包含：

- `requestId`
- `answerMode`，V0 固定为 `DETERMINISTIC`
- `matched`
- `fallback`
- `answer.title`
- `answer.sections`
- `evidence`
- `suggestedQuestions`

未命中问题返回 HTTP 200、`matched = false`、能力边界说明和推荐问题。问题为空、超过长度限制或项目 slug 格式非法返回 400。未知项目返回 404。未处理异常返回统一错误结构和 `requestId`，不得返回堆栈。

## 8. 确定性匹配和回答规则

问题规范化只做可解释转换：

- Unicode 规范化。
- 去除首尾空白。
- 合并连续空白。
- 统一英文大小写。
- 去除不影响语义的句末标点。

命中规则：规范化结果与规范问题或等价问法完全相等。V0 不做关键词打分和语义猜测，防止错误命中。

命中后的回答顺序固定为：

1. 背景
2. 职责
3. 技术方案
4. 验证过程
5. 最终状态

每段内容来自公开快照对应字段。Evidence 只返回项目引用且状态为 `APPROVED` 的条目。

## 9. 错误处理与韧性

- 快照反序列化失败、schema 版本不支持、引用缺失或状态枚举非法时，应用启动失败。
- API 使用统一错误响应，携带 `requestId`、错误码和用户可理解的信息。
- 前端保留用户输入并提供重试，不展示 Java 异常或内部路径。
- 首页或详情请求失败时展示局部错误状态，不渲染伪数据。
- V0 没有模型依赖，因此 DeepSeek 不可用不会影响 V0。
- 后续模型回答失败或流中断时，系统可调用同一个 `DeterministicAnswerEngine` 生成 `fallback = true` 的完整替代回答。

## 10. 安全与隐私

- 后端只读取审核后的公开快照，不读取 Obsidian 知识库。
- 不提供文件读取、搜索本地目录、上传、编辑或管理接口。
- 所有公共接口只读，回答接口不持久化访客问题。
- 对问题长度和 slug 格式做输入校验。
- 生产错误响应不泄露堆栈和环境信息。
- 构建产物不得包含候选快照、隐私扫描报告、原始截图或凭据。
- 提供自动隐私词扫描，至少检查 IPv4、Windows 绝对路径、内部 Linux 路径、密码字段和已知内部主机模式。
- Evidence 的 `APPROVED` 是公开返回的必要条件，`rawContentPublic = false` 时无原文访问入口。

## 11. 测试策略

实现遵循 TDD，每个业务行为先编写失败测试，再写最小实现。

后端测试覆盖：

- 快照正确加载。
- 非法 schema、重复 ID、悬空引用和未审核 Evidence 导致启动校验失败。
- 规范问题和等价问法正确命中。
- 未支持问题不会误命中。
- 五段回答顺序、项目状态、贡献类型和 Evidence 关联正确。
- 空问题、过长问题、非法 slug、未知项目和统一异常响应。
- API 集成测试验证三个公开端点。

前端测试覆盖：

- 首页加载和项目卡片导航。
- 详情页渲染项目内容。
- 推荐问题填入和提交。
- 命中回答的五段内容及 Evidence 卡片。
- 未命中提示。
- 加载、接口错误和重复提交保护。

端到端验收覆盖：

- 从首页进入 SQL 项目详情。
- 点击推荐问题并得到确定性回答。
- Evidence 卡片字段完整且不包含原始内部资料入口。
- 刷新项目详情路由仍由 Spring Boot 正确返回前端页面。
- 桌面和移动端核心路径可用。

## 12. V0 验收标准

切片完成必须同时满足：

1. Java 21 与 Maven 可以完整构建后端。
2. Node 与 npm 可以完整构建前端。
3. 后端、前端和端到端自动测试通过。
4. 可执行 JAR 中包含 Vue 静态产物。
5. 启动单个 JAR 后，首页、详情页和三个 API 可访问。
6. 规范问题返回五段式回答和一张已审核 Evidence 卡片。
7. 非支持问题返回边界说明，不伪造答案。
8. 自动隐私扫描没有命中禁止模式。
9. 页面通过响应式、键盘焦点、深浅主题和减少动画检查。
10. README 给出本地开发、测试、构建和运行说明。

## 13. V0 非目标

- DeepSeek、Spring AI 运行链路和 SSE。
- 多项目、多轮对话和会话记忆。
- 私有知识库在线查询。
- PostgreSQL、向量数据库和 Embedding。
- 登录、管理后台、内容编辑和访客问题留存。
- 公开原始日报、内部截图和内部文档。
- 模拟面试。
- 完整生产服务器、域名和 HTTPS 上线。V0 只交付可容器化、可部署的单制品及配置基础。

## 14. 后续扩展

V0 稳定后，按同一数据模型增加其余三个项目、12 个 FAQ 和更多 Evidence，再接入 DeepSeek。模型只能读取审核后的公开快照；模型失败时由确定性引擎提供完整 fallback。SSE 作为新增接口实现，不破坏 V0 REST 契约。
