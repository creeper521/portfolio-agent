# Agent 响应式作品窗口设计

> **状态：** 待用户书面评审  
> **日期：** 2026-07-23  
> **适用项目：** `D:\code\agent`  
> **范围：** 先调整 `design-exploration/warm-cream-compare.html`，视觉确认后再迁移到正式 Vue `/agent` 页面  
> **选定方向：** C「混合式响应布局」＋完整圆角作品窗口

## 1. 背景与设计判断

当前正式页面和暖调 Demo 已经共享相近的纸张色、暖色中栏与牛血红 CTA，但两者观感仍有明显差距。差距主要来自结构，而非继续微调颜色：

1. 正式页面默认三栏比例约为 `15% / 54% / 31%`，右侧证据栏过宽，削弱对话主舞台。
2. 正式页面铺满整个视口，缺少可感知的作品边界；在超宽或高视口上更像后台工具。
3. 深色全宽 Header 是页面最强对比，视觉入口落在网站外壳而非 Agent 内容。
4. 正式页面左右栏和中栏的水平留白偏大，真实内容不足时形成松散而非有意的编辑留白。
5. 对比截图使用了不同宽高比和不同内容状态，后续验收必须统一视口与数据，避免误把截图条件当成设计提升。

本轮选择混合式方案：大屏把 Agent 作为一件居中的作品陈列；中等屏逐渐贴近全屏；移动端恢复无外边距的工具布局。圆角属于整个 Agent 壳层，不给每个面板重复加圆角卡片。

## 2. 目标

1. 在超宽屏上形成和 Demo 一致的“深色舞台上的作品窗口”。
2. Header 与三栏工作区属于同一个完整圆角外壳，避免上下割裂。
3. 中栏稳定成为最宽、最强的视觉主角。
4. 保留现有会话、证据、拖拽分栏、抽屉、键盘和隐私行为。
5. 不引入通用 SaaS 卡片、玻璃拟态、渐变、霓虹或多层圆角容器。
6. 先在独立 Demo 中验证，再以最小风险迁移到 Vue 正式页面。

## 3. 选定布局

### 3.1 大屏：作品窗口

当视口宽度 `>= 1440px`：

- 页面外层使用深暖墨色舞台 `#2a2620`。
- Agent 壳层水平、垂直外边距为 `24–32px`。
- Agent 壳层最大宽度为 `1600px`，宽度不足时使用 `calc(100vw - 48px)`。
- 壳层圆角固定为 `16px`。
- Header 与三栏工作区共同位于壳层内，由壳层统一 `overflow: hidden`。
- 使用单层阴影 `0 30px 80px rgba(0, 0, 0, 0.5)` 表达作品与舞台的距离；不额外添加高对比描边。
- 工作区可用高度为视口高度减去舞台上下留边与壳层 Header 高度，不产生页面级纵向滚动。

推荐壳层几何：

```css
--agent-stage-gap: clamp(24px, 2vw, 32px);
--agent-shell-max: 1600px;
--agent-shell-radius: 16px;
```

### 3.2 中等屏：贴边窗口

当视口宽度在 `981–1439px`：

- 保留统一壳层和圆角语义。
- 外边距减至 `12–16px`。
- 圆角减至 `10–12px`。
- 不强制最大宽度，优先保证三栏或抽屉的现有响应式行为。
- `1220px` 以下仍按既有行为把证据栏切换为抽屉。

### 3.3 移动端：全屏工具

当视口宽度 `<= 980px`：

- 外层舞台留边归零。
- 壳层圆角归零。
- 页面恢复全屏工具形态。
- 会话栏和证据栏继续使用现有抽屉交互。
- 不为了保留桌面作品感而压缩手机内容宽度。

## 4. 壳层与 Header

### 4.1 统一圆角边界

圆角只施加在 Agent 壳层最外层。Header、会话栏、中栏和证据栏保持矩形内部结构，通过壳层裁切自然形成四个外角：

- Header 左上、右上继承壳层圆角。
- 会话栏左下、证据栏右下继承壳层圆角。
- 三栏内部边界继续使用 `1px` 规则线。
- 禁止给三栏分别增加圆角，避免出现拼装式卡片感。

### 4.2 Agent 路由 Header

Agent 路由的 Header 改为浅暖纸色，并纳入作品窗口：

- 背景基准使用 `#efe7d8` 或现有纸张令牌计算出的等价值。
- 文字使用深墨色。
- 当前 Agent 导航继续使用牛血红文字与底部细线。
- Demo Header 保持现有 `60px`；正式 Vue 迁移保持项目现有 `66px`。
- 非 Agent 路由的 Header 主题不受影响。

深墨色不再作为 Agent 页全宽 Header 背景，而保留给新对话、用户消息、必要的主动作和外层舞台。

## 5. 三栏比例与分栏约束

### 5.1 默认宽度

桌面默认值调整为：

| 区域 | 默认宽度 | 角色 |
|---|---:|---|
| 会话栏 | `250px` | 导航与会话管理 |
| 对话栏 | 剩余空间 | 绝对主舞台 |
| 证据栏 | `340px` | 辅助核验 |

在 `1600px` 壳层中，对应比例约为 `15.6% / 63.1% / 21.3%`，接近 Demo 的 `17% / 62% / 21%`。

### 5.2 可调整边界

为避免证据栏再次占到接近三分之一，正式实现时更新分栏契约：

```text
会话栏：220–320px
证据栏：300–420px
对话栏：最低 640px
```

用户仍可通过 Pointer、键盘、Shift 加速、Home 和双击调整或复位。保存键 `portfolio.workspace.split.v1` 保持不变；读取旧值时必须按新边界 clamp，不能产生布局溢出。

这一条会取代 `docs/04-项目代码约束.md` 当前的 `会话 220–380 / 证据 380–760 / 对话最低 600` 视觉布局数值。正式迁移前必须同步更新权威约束与测试。

## 6. 密度与视觉层级

### 6.1 栏内留白

目标基准：

| 区域 | 当前 | 目标 |
|---|---:|---:|
| 会话栏水平 padding | `23px` | `18–20px` |
| 对话 Header 水平 padding | `35px` | `24–28px` |
| Composer 左右定位 | `35px` | `24–28px` |
| 证据栏水平 padding | `25px` | `18–20px` |

垂直间距可继续沿用现有节奏，重点先收紧水平结构。正文阅读宽度仍受限制，不因中栏变宽而无限增长。

### 6.2 对比度分配

- 深色舞台：建立外部作品边界。
- 浅暖 Header：把注意力交还给 Agent 内容。
- 暖燕麦中栏：主要阅读面。
- 深墨用户消息：局部对话焦点。
- 牛血红发送按钮：唯一强色 CTA。
- 红色细线：当前会话、当前证据和导航状态。
- 其他操作继续使用透明背景、规则线和文字状态。

## 9. 验收方式

### 9.1 同条件对比

后续所有对比必须同时满足：

- 相同视口。
- 相同会话与证据数据。
- 相同对话状态，至少包含两轮用户问题和一轮 Agent 回答。
- 相同浏览器缩放。

不得再直接比较 `1882×1977` 空状态与 `2094×1356` 填充状态。

### 9.2 Demo 视觉验收

至少检查：

```text
2048 × 1080
1440 × 900
1219 × 900
980 × 800
390 × 844
```

通过条件：

1. `>=1440px` 可明确感知深色舞台和完整圆角作品窗口。
2. Header 与工作区属于同一个壳层，四角裁切自然。
3. 中栏约占壳层六成，右栏不再抢主焦点。
4. 内部三栏没有重复圆角或浮卡感。
5. `981–1439px` 留边减少但不出现横向滚动。
6. `<=980px` 恢复全屏，不浪费移动端空间。
7. A/B/C/D 和 CTA 切换继续正常。
8. 控制台无错误。

## 10. 非目标

- 不把整个站点都放进深色舞台。
- 不改变首页、项目页、时间线或证据中心的 Header。
- 不给每张证据、每条消息或每个面板增加大圆角。
- 不引入玻璃拟态、装饰渐变、绿色状态色或通用 Dashboard 卡片。
- Demo 阶段不实现正式分栏拖拽与抽屉无障碍逻辑。
- 未经 Demo 视觉确认，不直接迁移正式 Vue 页面。
