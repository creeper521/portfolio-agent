## 1. 结论

当前前端的路由、公开数据边界、本地会话、Agent 分栏和测试基础可以保留，但视觉实现偏离了目标原型。首页被重新设计成深色编辑风格，身份区、探索区和 Agent 内容密度也发生了结构性变化。

本轮不重写业务架构，采用“保留功能骨架、重做呈现层”的方式收口：

1. 目标原型是首页和 Agent 工作台的唯一视觉基线。
2. 多路由页面继续存在，但必须继承同一套纸张档案语言。
3. 不复制原型中的姓名占位符、示例数量或虚构证据。
4. 现有公开数据、路由查询参数、本地会话和分栏持久化继续作为事实与状态来源。
5. 验收以同视口截图对照、交互测试和可访问性行为共同完成。

## 3. 全局视觉规则

### 3.1 颜色职责

核心令牌保持：

```css
--paper: #f4eee4;
--paper-hi: #fbf7ef;
--paper-low: #e9dfd0;
--ink: #201c17;
--ink-2: #34302a;
--muted: #746b5e;
--faint: #9c9182;
--rule: #cdbfa9;
--red: #7a2e2a;
--red-hi: #b65d53;
```

使用规则：

- 首页和普通内容页以 `--paper` 为主背景。
- `--ink` 只用于回答舞台、Agent 对话区、项目重点章节和 hover 反色。
- `--red` 只用于批注、当前状态、引用、细线和关键文字动作。
- 不使用整块暗红作为身份、证据或导航选中背景。
- 不增加绿色、青色、蓝色、紫色或装饰性渐变。

### 3.2 排版与宽度

- 主内容宽度恢复到 `1180px`。
- Header 高度保持 `66px`。
- 中文标题和长回答使用衬线体。
- 导航、按钮、说明使用无衬线体。
- 编号、证据代码、日期和状态使用等宽体。
- 首页 Hero 标题桌面最大约 `92px`，手机约 `62px`。
- 长正文控制在 `680–820px`。

### 3.3 交互与动效

- Hero 副标题使用一次性进入动画。
- 统计项和探索入口 hover 时最多上移 `6–8px`。
- 推荐问题 hover 通过缩进、文字色和浅底色表达。
- 首页回答允许前端逐字呈现，持久化时只保存完整消息。
- `prefers-reduced-motion: reduce` 下取消逐字、错峰和位移动画。

## 4. 首页设计

### 4.1 Header

首页 Header 使用浅纸色半透明背景、细规则线和模糊效果。左侧显示可用的作者标识；姓名为空时显示“工程成长档案”。右侧首页锚点为“概览 / 对话 / 探索”，随后显示当前身份和“完整 Agent”。

其他路由继续显示多路由导航，但视觉保持浅色。Agent 路由切换为深色 Header。

移动端保留作者标识、“完整 Agent”和目录按钮。目录展开为纸张目录层，支持 Escape 关闭和焦点恢复。

### 4.2 Hero

首页 Hero 恢复浅纸布局：

- 顶部横向卷宗索引。
- `ENGINEERING DOSSIER / 00` 章节标签和暗红短线。
- 主标题与暗红副标题形成两行构图。
- 一段衬线说明文字。
- “开始浏览”和“直接提问”文本按钮。
- 底部扫描线和 `SCROLL TO INSPECT`。
- 轻微径向纸色变化和纵向网格纹理。

姓名为空时，使用真实职位作为主标题，使用“工程实践档案”作为副标题，不显示姓名占位符。

### 4.3 可信度摘要

先展示章节介绍，再展示真实统计。统计由项目、证据、时间线和公开周期派生，值为零的项不渲染。整个统计格可点击，hover 使用浅纸高光与轻微上移。

### 4.4 身份轻对话

控制台由左侧身份区和右侧问题区构成：

- 身份按钮为紧凑 2×2 网格。
- 当前身份使用深墨色背景。
- 身份说明单独显示在网格下方。
- 每个身份提供 4 个推荐问题。
- 推荐问题显示 `Q·01` 编号和右箭头。
- 底部提供自定义问题输入。

选择问题后，在控制台下方展开深色回答舞台：

- 左侧显示身份代码、轮次和回答模式。
- 右侧显示问题、逐字回答、完成状态、引用和操作。
- 操作为“继续追问”“查看相关项目”“带着上下文进入 Agent”。
- 首页最多承载 3 轮，超过后引导进入 Agent。

### 4.5 探索和页脚

探索区保持浅纸背景，四个入口只在 hover 时局部反色。页脚恢复 About、Contact 和 Privacy 三列；缺失联系方式时隐藏对应项。

## 5. Agent 上下文交接

首页进入 Agent 时传递：

```ts
interface AgentRouteSeed {
  role: AudienceRole
  question: string
  answer: string
  projectSlug: string | null
  evidenceIds: string[]
  source: 'HOME' | 'PROJECT' | 'EVIDENCE'
}
```

查询参数只承担可分享的身份、问题、项目和单条证据定位。首页已生成的摘要通过 `history.state` 或会话初始化参数传递；刷新后若状态不存在，则由公开数据重新生成确定性摘要。

新会话初始化必须幂等：同一 seed 不得重复插入消息。

## 6. Agent 工作台

### 6.1 布局

桌面继续使用：

```text
会话栏 | 对话栏 | 证据栏
```

可见分隔线为 1px，命中区域为 12px，不在列之间产生明显黑色沟槽。

### 6.2 会话栏

- 顶部只保留紧凑的新建会话按钮。
- 会话使用规则线列表。
- 当前会话使用暗红左边线。
- 删除动作默认弱化，在 hover 或焦点时增强。
- 隐私说明位于底部。

### 6.3 对话栏

- 空状态不使用巨型边框卡片。
- 用户和 Agent 消息都使用左侧细线与自然文本流。
- 用户消息不使用实心矩形背景。
- 引用紧跟回答，点击后打开或定位右侧证据。
- Composer 固定在底部，保持约 `58px` 高度。

### 6.4 证据栏

- 顶部提供“证据 / 项目 / 时间线 / 脉络”标签。
- 当前引用使用浅底和暗红左边线。
- 列表卡显示证据代码、标题、摘要和元数据。
- 点击消息引用后滚动到对应条目并短暂强调。

### 6.5 抽屉

- `<1220px`：证据栏为右抽屉。
- `<=980px`：会话栏也为左抽屉。
- 抽屉互斥，具有遮罩、Escape 关闭、焦点约束和焦点恢复。
- 关闭抽屉时对隐藏面板设置 `inert` 和 `aria-hidden`。

## 7. 其他路由

### 7.1 项目目录

保留纵向案卷索引，缩小页头高度和标题，使用浅纸背景。技术标签和状态为辅助信息，项目标题与问题摘要为主要信息。

### 7.2 项目详情

保留现有章节，调整深色封面比例、正文宽度和移动端标题断行。深色方案章节作为单一重点舞台，不扩大成独立主题。

### 7.3 时间线

页头改为浅纸样式。时间轴保持编号、日期和决策正文的关系，关键节点可使用局部深色强调。

### 7.4 证据中心

选中项不再使用整块暗红。证据详情减少卡片感，优先使用纸张底色、规则线和左侧批注线。移动端状态必须横向可读。

## 8. 可访问性

- 所有身份、问题、入口、引用和导航支持键盘。
- 页面路由切换后焦点移至主标题。
- 输入和按钮具有可见焦点。
- 抽屉打开后焦点不能落到背景控件。
- 分隔条保留 ARIA separator、方向键、Shift 步长、Home 和双击复位。
- 截图只能确认视觉风险，最终仍需键盘和 DOM 测试。
