# Portfolio Agent 浅色工作台配色收口设计

> **状态：** 已完成视觉方向讨论，待用户审核本文后进入实施计划
> **日期：** 2026-07-22
> **适用项目：** `D:\code\agent`
> **范围：** `/agent` 三栏工作台的浅色主题收口
> **选定方向：** A「均衡纸阶」

## 1. 背景与结论

当前 Agent 工作台采用“浅色会话栏、深色对话区、浅色证据栏”的三栏结构。中间深色区域占据页面近一半宽度并贯穿整个视口，在空状态和长对话场景下形成过重的深色平面，降低正文可读性，也使左右两栏退化成附属区域。

本轮采用 A「均衡纸阶」：顶部导航继续使用深墨色，三栏工作台统一进入米白纸张体系；中间对话区最亮，左右辅助区域各降低一档。深色实心按钮只保留“新对话”和“发送”，暗红色只承担当前状态、引用、身份和关键文字动作。

本轮不修改布局、数据、交互或业务能力，只调整 Agent 路由的颜色职责与组件状态。目标是提高长内容阅读舒适度，同时保留现有工程档案的编辑感和视觉识别度。

## 2. 设计目标

1. 消除贯穿全高的大面积深色对话舞台。
2. 让中间对话区成为最清晰、最适合阅读的视觉中心。
3. 使用同一纸张色系建立三栏层级，不把页面处理成完全同色的平面。
4. 把深墨色和暗红色限制在真正需要强调的控件与状态上。
5. 保持桌面分栏、响应式抽屉、键盘操作和隐私提示的现有行为。
6. 不引入新的颜色家族、装饰性渐变、阴影卡片或大面积暗红背景。

## 3. 选定方案与备选方案

视觉讨论比较了三种浅色方向：

- A「均衡纸阶」：中间最亮，左右轻微降一档。阅读焦点明确，区域之间没有明显拼接感。**本轮采用。**
- B「单一米白」：三栏完全同色，只靠规则线区分。最安静，但主区与辅助区的层级不足。
- C「档案分层」：侧栏更深，卡片底色更明显。档案感最强，但视觉重量偏高，接近传统后台工作台。

A 在阅读舒适度、三栏层级和品牌延续之间最均衡，因此作为正式实施基线。

## 4. 颜色职责

### 4.1 全局颜色保持不变

继续使用现有令牌：

```css
--paper: #f4eee4;
--paper-hi: #fbf7ef;
--paper-low: #e9dfd0;
--ink: #201c17;
--ink-2: #34302a;
--muted: #746b5e;
--faint: #9c9182;
--rule: #cdbfa9;
--red: #7a2e2a;
--red-hi: #b65d53;
```

不得新增蓝色、绿色、青色、紫色、渐变或与现有纸张体系无关的中性色。

### 4.2 Agent 工作台语义色

在 `.agent-workspace` 作用域内建立语义令牌，组件不再直接散落十六进制颜色：

```css
--workspace-header-bg: var(--ink);
--workspace-rail-bg: color-mix(in srgb, var(--paper) 72%, var(--paper-low));
--workspace-thread-bg: var(--paper-hi);
--workspace-evidence-bg: var(--paper);
--workspace-surface-subtle: color-mix(in srgb, var(--paper-low) 46%, transparent);
--workspace-text: var(--ink);
--workspace-text-secondary: var(--muted);
--workspace-text-faint: var(--faint);
--workspace-rule: var(--rule);
--workspace-accent: var(--red);
--workspace-accent-soft: var(--red-hi);
--workspace-primary-bg: var(--ink);
--workspace-primary-text: var(--paper-hi);
```

`color-mix()` 只组合现有纸张令牌，不引入新的独立色板。若目标浏览器兼容性检查不通过，应使用经过现有令牌计算后固化的单一回退值，并在测试中记录。

### 4.3 区域分配

- 顶部导航：`--workspace-header-bg`，继续作为全页唯一的大面积深色。
- 左侧会话栏：`--workspace-rail-bg`，比右栏略深，承担导航与管理职责。
- 中间对话区：`--workspace-thread-bg`，使用最亮纸色，承担主要阅读职责。
- 右侧证据栏：`--workspace-evidence-bg`，比对话区略暗，承担辅助核验职责。
- 三栏分隔：`1px --workspace-rule`；可拖动命中范围仍为现有宽度，不形成深色沟槽。
- 当前会话、当前证据和局部引用：允许使用 `--workspace-surface-subtle`，但不得形成悬浮卡片或整栏色块。

## 5. 控件与内容层级

### 5.1 深色主按钮

只有以下两个控件使用实心深墨背景：

1. 左栏“新对话”。
2. 中栏 Composer 的“发送”。

主按钮默认使用 `--workspace-primary-bg` 与 `--workspace-primary-text`。Hover 只允许轻微调整明度；Focus 使用清晰的暗红轮廓；Disabled 必须同时降低背景和文字对比，但仍保持标签可读。

### 5.2 次级操作

以下操作使用透明或纸色背景、规则线边框及深墨/暗红文字，不得使用实心深色：

- 模式标签；
- 继续追问；
- 查看证据；
- 查看项目或时间线；
- 重试回答；
- 清空当前对话；
- 会话删除；
- 抽屉开关。

危险操作以暗红文字和可见焦点表达，不使用整块暗红背景。

### 5.3 对话消息

- 用户问题：自然文本流，使用 `2px --workspace-accent` 左线，不使用实心消息气泡。
- Agent 回答：自然文本流，使用 `1px --workspace-rule` 左线；正文使用 `--workspace-text`。
- 身份、回答来源、验证状态和引用编号：使用等宽体与暗红色，但字号不得小到影响识别。
- 引用和追问操作紧跟回答，不建立独立深色工具条。
- 空状态大标题使用深墨色，不再反白显示。

### 5.4 Composer

- Composer 使用 `--workspace-thread-bg` 或相邻纸色背景与 `1px --workspace-rule` 边框。
- 输入文字使用深墨色，占位文字使用 `--workspace-text-faint`。
- “发送”是 Composer 中唯一实心深色元素。
- Pending、Error 和 Disabled 状态不得通过大面积深色反转表达。

## 6. 组件修改边界

### 6.1 `AgentWorkspace`

- 定义工作台语义色。
- 把工作台根背景从深墨色改为纸张体系。
- 保持三栏宽度、分隔条、抽屉状态和隐私提示位置不变。
- 桌面分隔条只显示细规则线；命中区域继续满足拖动与键盘操作。

### 6.2 `ConversationThread`

- 完成深色到浅色的整体反转。
- 更新标题、空状态、用户消息、Agent 消息、回答 section、引用、次级按钮、Composer、加载、错误和禁用状态。
- 保留现有 DOM 结构、消息顺序、ContextEnvelope 操作和 API 行为。

### 6.3 `LocalSessionRail`

- 使用左栏暖米色背景。
- 当前会话使用暗红左线与很淡的纸张底色。
- “新对话”保留为深色实心按钮。
- 删除动作、清空动作和隐私提示继续弱化显示，但必须满足可读性与焦点要求。

### 6.4 `EvidenceDesk`

- 使用右栏纸色背景。
- 当前引用使用暗红左线与局部浅底。
- 标签、项目、时间线和脉络视图继续通过规则线与文字状态区分，不使用深色 tab 背景。

### 6.5 `AgentPage`

- 路由加载、handoff 和失败反馈状态改用浅色对话区语义，不再显示整屏深色背景。
- 不改变路由 seed、一次性 handoff 或错误重试逻辑。

## 7. 响应式与抽屉

- 桌面三栏采用选定的 A「均衡纸阶」。
- `<1220px` 时右侧证据抽屉继续使用 `--workspace-evidence-bg`。
- `<=980px` 时左侧会话抽屉继续使用 `--workspace-rail-bg`。
- 抽屉遮罩是唯一允许保留的临时半透明深色覆盖层；遮罩关闭后页面不得残留深色平面。
- 抽屉的 Escape、焦点约束、焦点恢复、`inert` 和 `aria-hidden` 行为保持不变。
- 移动端 Composer、抽屉按钮和关键状态必须保持清晰，不因浅色反转出现低对比文字。

## 8. 状态与无障碍

- 正文、标题、按钮和表单文字应满足 WCAG AA 对比度要求。
- Focus-visible 使用暗红轮廓或暗红加深墨的双重信号，不能只依赖底色变化。
- Hover 不是当前会话、当前证据或当前 tab 的唯一识别方式。
- Pending 使用文字和轻微状态标识，不使用闪烁或低对比骨架。
- Error 使用暗红文字、规则线和明确的“重新回答”操作，不使用整块红底。
- `prefers-reduced-motion: reduce` 下所有内容仍立即可见。
- 分隔条的 ARIA、方向键、Shift 步长、Home 和复位行为不受配色修改影响。

## 9. 测试与验收

### 9.1 自动化测试

- 更新视觉契约测试，确认 Agent 工作台不再以 `--ink` 作为三栏根背景。
- 确认只有“新对话”和“发送”使用实心深色主按钮语义。
- 保留并运行 `AgentWorkspace`、`ConversationThread`、`AgentPage`、会话和分栏行为测试。
- 验证 loading、error、pending、disabled、boundary、rejected、retrieval、model 和 fallback 状态在浅色主题下仍使用正确标签与操作。
- 保留 reduced-motion、键盘分隔条、抽屉和页面内存会话测试。

### 9.2 视觉验收

使用真实公开内容检查以下视口：

```text
1440 × 900
1219 × 900
980 × 800
390 × 844
```

验收标准：

1. 顶部导航是唯一的大面积深色区域。
2. 中间对话区明显比左右栏更亮，但三栏仍属于同一纸张体系。
3. 页面不存在贯穿全高的深色对话平面。
4. “新对话”和“发送”是仅有的实心深色主按钮。
5. 用户与 Agent 消息无需背景卡片即可清晰区分。
6. 当前会话、当前证据、引用和 tab 状态可通过颜色之外的线条或文字共同识别。
7. Composer、抽屉、错误和加载状态不存在低对比文字。
8. 无横向溢出，分栏与抽屉行为没有回归。

## 10. 非目标

- 不修改后端 API、回答契约或公开内容。
- 不改变三栏布局、宽度约束、拖动、抽屉或会话行为。
- 不重写消息 DOM，不增加消息气泡、头像或新的图标体系。
- 不新增路由、状态库、UI 框架或动画库。
- 不调整顶部导航的结构与深色视觉。
- 不顺带重排项目、时间线、证据和首页。
- 不扩大 C1/C2/C3 能力，不修改隐私和持久化边界。
